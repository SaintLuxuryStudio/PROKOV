# Как опубликовать сайт в интернет

Одним абзацем: проект — Next.js (Node/React), его можно разместить как на полностью управляемых платформах (Vercel/Netlify), так и на собственном VPS/хостинге; ниже — что это значит, что выбрать, и как действует ИИ.

## Как работает размещение
- Ваш код (Next.js) превращается в сборку (статические файлы + серверные функции), которую нужно обслуживать через HTTP.
- Хостинг даёт:
  - Домен (адрес типа mysite.com) или возможность привязать ваш.
  - Сервер/функции, где крутится Node и отдаются страницы/API.
  - CDN/кэш, SSL-сертификат, логирование, масштабирование.
- Деплой — это процесс доставки кода на хостинг (обычно через git push или загрузку артефакта), после чего платформа собирает и запускает приложение.

## Что выбрать
1) **Vercel** (рекомендация по умолчанию для Next.js):
   - Плюсы: нативная поддержка Next.js (app router), автоматический билд, превью на каждый PR, бесплатный тариф на старте, встроенный CDN/SSL.
   - Минусы: бесплатный тариф ограничен по функциям/лимитам; платно — по факту использования.
2) **Netlify**:
   - Плюсы: простая статика/SSR, есть Edge Functions, бесплатный старт, SSL/CDN.
   - Минусы: для сложного SSR иногда больше ручных настроек, чем на Vercel.
3) **Render/Fly.io/VPS (DigitalOcean/Linode/etc.)**:
   - Плюсы: полный контроль (версии Node, процессы, фоновые задачи), предсказуемая цена для маленьких инстансов.
   - Минусы: больше ручных настроек (PM2/systemd, nginx), обновления и безопасность на вас.

## Быстрый рецепт (Vercel)
1. Установите Vercel CLI: `npm i -g vercel` (или используйте web-интерфейс).
2. В корне проекта: `vercel` → войдите в аккаунт → выберите автодетект Next.js.
3. Vercel создаст проект, соберёт `npm install && npm run build`, выдаст временный домен.
4. Привяжите свой домен в консоли Vercel → обновите DNS (A/ALIAS/ANAME или CNAME). SSL выдаётся автоматически.

## Быстрый рецепт (Netlify)
1. В Netlify: "Add new site" → "Import from Git" → подключите репозиторий.
2. Build command: `npm run build`; Publish directory: `.next` (для Next 15 включить Netlify Next Runtime, автоопределится).
3. Получите домен вида `*.netlify.app`, при необходимости подключите свой домен (CNAME/ALIAS), SSL авто.

## Ручной деплой на VPS (кратко)
1. Взять VPS (Ubuntu), поставить Node LTS, nginx, certbot.
2. Скопировать код или git pull, `npm ci`, `npm run build`.
3. Запуск: `npm run start` за reverse-proxy nginx (или PM2 для автоперезапуска). Nginx отдаёт 80/443 → проксирует на локальный порт (например, 3000).
4. Настроить SSL через certbot (Let’s Encrypt) и автообновление.

## Может ли это сделать ИИ?
- ИИ может подготовить конфиги (nginx, PM2), команды, файлы `vercel.json` или `netlify.toml`, настроить DNS-инструкции, а также провести вас через CLI-процедуры.
- ИИ **не** сможет сам щёлкать кнопки в личном кабинете или вносить DNS-записи, но может дать точные значения (какие записи, куда вставить) и проверить ваши логи/ошибки.

## Что нужно решить заранее
- Какой домен использовать и у какого регистратора он находится.
- Бюджет и желаемый уровень автоматизации: "минимум действий" → Vercel/Netlify; "полный контроль" → VPS.
- Требуются ли фоновые задачи/вебхуки/база данных — от этого зависит выбор тарифа и типа хостинга.

## Минимальный чек-лист для вас
- [ ] Выбрать платформу (Vercel / Netlify / VPS).
- [ ] Проверить Node LTS (>=18) и переменные окружения, если нужны.
- [ ] Настроить Git-репозиторий для автодеплоя (или CLI Vercel/Netlify).
- [ ] Привязать домен и обновить DNS.
- [ ] Убедиться, что сборка `npm run build` проходит локально.
- [ ] Проверить работу сайта по HTTPS после деплоя.
